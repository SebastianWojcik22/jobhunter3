Prompt Engineering — Core Work

Anti-Hallucination Prompt Architecture (RiseWay3)
Designed a production prompt system for GPT-4o to analyze 113 EU AI Act articles — each prompt injects the full article verbatim between delimiters, preventing the model from drawing on prior knowledge. Built a versioned prompt library covering: plain-language summarization, compliance category classification (8 categories), risk level scoring, and multi-article batch processing. System processes the entire EU AI Act and syncs structured output to Notion.
Multi-Variant Selection Prompt (JobHunter3)
Engineered a prompt that receives 6 different CV profile variants simultaneously and instructs GPT-4o to: (1) select the best-matching CV for a given job offer, (2) provide a match score with reasoning, and (3) detect dealbreakers (e.g. "5+ years experience required", "mandatory CS degree"). All output is structured JSON with strict schema validation. Replaces 6 separate API calls with 1, maintaining accuracy while cutting cost and latency.
Iterative Quality Control Loop
Built an automated reprocessing pipeline: articles/offers that score below a confidence threshold are flagged and re-queued with a refined prompt variant. Implemented prompt A/B testing to measure output consistency across runs. Result: <3% hallucination rate on regulatory content, verified against source documents.
